# Active Step recovery plan

## Goal

Make paused Step work visible and recoverable without weakening the runner's safety checks.

## Acceptance criteria

1. Any unresolved run is named in the main World shell.
2. The creator can open that Step or stop it safely when cancellation is allowed.
3. Add Experience explains the same recovery path.
4. Forge-owned presentation metadata cannot make an unchanged approved game build look stale.
5. Real game files, planning records, Git state, remotes, and approved scope remain protected.
6. Focused, full, protected compatibility, build, type, context, and browser checks pass.

## Scope

Dashboard recovery UI, plain error copy, the controlled-inventory boundary for Forge presentation metadata, tests, and concise operational documentation only.
