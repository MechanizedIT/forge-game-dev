# v0.1.0 First-Run Usability Plan

- **Workflow state:** `COMPLETE`
- **Baseline:** `a5321ed1406c599de70a9ba951552d81aee82e7a`
- **Source:** Direct feedback from a real first-time judge-path run

## Scope

Clarify the existing Enemy Targeting experience without adding another workflow or product area: orient the World screen, explain preserved workspaces, expose the existing safe reset in the dashboard, make unavailable navigation honest, strengthen long-running progress, and clarify game controls.

## Acceptance criteria

- A first-time user can identify Forge, the five-step demo, the starting action, the three-file change, and the proof model from the first World screen.
- Preserved setup reports `available`, `in progress`, or `completed` and prints the exact reset command.
- A completed dashboard offers **Start fresh demo** behind explicit confirmation and delegates to the existing reset function.
- Proof and Chronicle are disabled until real evidence exists.
- Active Codex work prominently shows the current and completed stages, plain-language activity, elapsed time, a several-minute expectation, and optional technical detail.
- The baseline player and headless verifier support arrow keys and WASD; pre-launch instructions state the exact play sequence.
- Existing SDK, scope, verification, confirmation, persistence, CLI, and fixture-copy guarantees remain intact.
