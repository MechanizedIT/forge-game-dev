- All portable artifacts are strict-schema JSON plus deterministic Markdown where required; every record round-trips through its schema and contains no absolute local path.
- The committed starter inventory is fixed and contains no external asset, audio, font, dependency, plugin, download, or generated package.
- The fixed Godot command proves the project/main scene/scripts/input/movement/required nodes and emits one stable Forge-owned success marker; failure prevents promotion and registration.
- The local Git baseline stages only the expected project tree, uses a project-local Forge identity, records `HEAD`, and proves a clean worktree without touching remotes or this repository.
- Failed or cancelled work records a concise safe failure, removes only the current transaction's verified staging directory, and never invokes sample reset or removes another project.
- A completed project is registered only after promotion, reloads after host restart, appears in Launchpad recent projects, and reopens the honest Project Created summary with Project World labeled pending.
