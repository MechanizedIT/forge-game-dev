# Creator workflow repair plan

## Goal

Remove the loops and dead ends found during the real Signal Sweep creator test without replacing Forge's planner, runner, verification, recovery, or architecture boundaries.

## Plan and acceptance

1. Keep the global active-Step banner everywhere except that Step's Detail and Work pages, where status and safe actions are shown inline.
2. Make the center workspace independently scrollable at desktop and 390×844, including long failure and Advanced Details content.
3. Derive approved, running, failed, playtest, feedback, completed, and stopped actions from the existing runner snapshot.
4. Validate each Add Experience field separately, preserve all field text, and shorten the visible recommendation flow.
5. Launch the existing Godot project directly from Test, then persist contextual feedback after Godot closes.
6. Route Needs a Change and Broken into linked, file-aware follow-up preparation and restore real Step editing.
7. Protect the existing backend and registered game state with focused tests, one full suite, compatibility/build checks, and read-only browser QA.

## Scope

The existing dashboard presentation, planner request limits, project presentation history, Godot launch timing, and follow-up routing only. No new workflow engine, runner, Repair engine, notes platform, dependency, backend hierarchy, or commit.
