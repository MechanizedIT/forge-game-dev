- The page includes every required section, the exact hero message, both walkthroughs with no more than seven steps, current/future separation, architecture, proof, local setup, submission links, and final calls to action.

- The persistent walkthrough label and plain-language local-only explanation prevent fixture-backed states from appearing live.
- Link configuration never renders an absent optional URL as an active link; no local video, credentials, private plan URL, or `/feedback` identifier reaches public assets.
- Evidence validation catches missing files, missing source references, unsafe paths, unsupported claims, stale version labels, decorative/evidence confusion, and accidental current/future boundary drift.
- Keyboard, touch, focus, live announcements, disclosures, dialog behavior, reduced motion, responsive layouts, contrast, overflow, console, and network-independence checks pass at `1440×900`, `768×900`, and `390×844`.
- Production budgets cover JavaScript, CSS, initial assets, and the largest public image; no operational Forge module is bundled.
- `npm run context:check`, `npm run check`, the main production build, `npm run check:v0.1`, and `git diff --check` pass at final verification.
- `npm run forge` still targets v0.2; `v0.2.0` remains at `08cffa7`; no deployment, push, merge, release, tag movement, serverless function, secret, analytics, or operational workflow change occurs.
